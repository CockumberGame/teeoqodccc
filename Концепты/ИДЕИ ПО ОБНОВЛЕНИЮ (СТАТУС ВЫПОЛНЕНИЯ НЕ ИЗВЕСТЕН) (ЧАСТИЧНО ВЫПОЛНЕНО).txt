   - staminaDepleted (stamina <= 0)
   - complication (событие)

   - Фаза calm/interested → карты осмотра, teasing

   - Квартира с кликабельными объектами
   - Кровать → восстановление stamina
   - Полки → инвентарь/магазин
   - Дверь → начало ночи

   - Нужно для магазина между ночами

   - Предметы модифицируют действия (staminaCost -2)
   - Нужно для магазина

clientsTonight = baseClients + reputationBonus + randomBonus

старт = 3 клиента

Это отражает усталость и нервное напряжение персонажа.

7. Система arousal и фаз

Система arousal остаётся числовой (0-100), но добавляются фазы возбуждения:

0–15 calm
15–35 interested
35–60 excited
60–80 ready
80–95 edge
95–100 climax

Фазы влияют на:

вероятность завершения

награду

риск ошибки

8. Система edge tension

Если arousal превышает 80:

активируется механика edge tension.

Каждое действие увеличивает:

tensionLevel

Эффекты:

награда увеличивается
риск ошибки увеличивается

Это создаёт риск-ревард стратегию.

Игрок может:

безопасно завершить

или продолжать ради большего reward.

9. Система раннего завершения

Если arousal > 70, игрок может попытаться завершить interaction раньше.

Результаты:

успех:

interaction заканчивается

награда ниже

неудача:

падает satisfaction

падает репутация

Это даёт стратегический выбор.

10. Система предпочтений клиента

Каждый клиент имеет предпочтения:

liked
neutral
disliked

Действия, совпадающие с liked:

увеличивают arousal

увеличивают reward

Disliked:

уменьшают эффективность

увеличивают шанс ошибки

11. Система скрытых traits

Каждый клиент имеет 1-3 скрытых traits.

Примеры:

generous
impatient
shy
demanding
curious
loyal

Traits изменяют:

скорость arousal

награды

реакции

Traits не раскрываются сразу.

Игрок может:

угадывать

или открывать их через навыки.

12. Навык проницательности

Навык perception или visual_assessment должен:

повышать шанс раскрытия traits

раскрывать предпочтения

давать подсказки о поведении клиента.

13. Система satisfaction

После interaction рассчитывается показатель:

satisfaction

Факторы:

достигнутый уровень arousal

совпадения предпочтений

количество ошибок

длительность interaction.

Satisfaction влияет на:

tips
reputation gain
return chance
14. Репутация

Репутация должна расти медленно.

Формула:

reputationGain = satisfaction * 0.2

Но есть лимит:

maxReputationPerNight = 25

Репутация влияет на:

количество клиентов

шанс редких модификаторов

вероятность особых клиентов.

15. Система настроения клиента

Во время interaction настроение клиента может меняться со временем.

Пример:

neutral → impatient → frustrated

Если interaction затягивается.

Это влияет на:

reward

риск ошибки

вероятность ухода клиента.

16. Архитектурные требования

Все новые системы должны быть реализованы модульно.

Следующие элементы должны храниться в конфигурационных данных:

traits

preferences

client modifiers

события

archetypes клиентов.

GameEngine должен только оркестрировать системы, а не хранить контент внутри кода.

Это позволит масштабировать игру и добавлять новый контент без изменения логики.
