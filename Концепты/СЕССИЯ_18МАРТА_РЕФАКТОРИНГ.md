## 🔍 Выявленные проблемы

### 1. **Размазанное состояние (State Management)**
```
GameEngine (state) → UI (дублирование) → EventBus (синхронизация)
```
- Multiple sources of truth
- UI не знал когда обновляться
- Данные рассинхронизированы

### 2. **Отсутствие единого источника правды**
- `gameState`, `actionState`, `hubState` в GameEngine
- Local state в React компонентах
- Синхронизация через EventBus (ненадёжно)

### 3. **Нарушение одностороннего потока данных**
```
Было: State → Render → Event → State Update → ??? (кто триггерит re-render?)
Стало: State → Render → Event → State Update → Re-render (через Context)
```

## 💡 Архитектурные принципы

### **Единый источник правды:**
```
GameEngine → GameContext → UI
```

### **Односторонний поток:**
```
Action → dispatch → reducer → state → render
```

### **Разделение ответственности:**
- **GameEngine:** бизнес-логика, правила, расчёты
- **GameContext:** синхронизация, состояние, события
- **UI:** отображение, ввод пользователя
