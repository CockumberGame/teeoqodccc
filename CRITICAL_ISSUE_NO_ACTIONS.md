## 🎯 КОРЕННАЯ ПРИЧИНА (ВЫЯВЛЕНА)

### Проблема: Мутация объекта без создания копии

**Файл:** `src/engine/GameEngine.js`  
**Метод:** `getActionState()`

```javascript
// ❌ ПРОБЛЕМА: Возвращаем ссылку на объект
getActionState() {
    return {
        // ...
        actionUsage: this.actionDeckSystem.currentClientUsage  // ← ССЫЛКА!
    };
}
```

**Что происходит:**

1. Клиент 1: `actionUsage = { hand_soft: 3, bj_soft: 2 }`
2. UI получает **ссылку** на этот объект
3. Переход к клиенту 2: `resetClientUsage()` → `currentClientUsage = {}`
4. **НО:** UI всё ещё держит **старую ссылку** в памяти React
5. React не видит изменений → не перерисовывается
6. **Результат:** `actionCards` в UI = `[]` (пустой массив)

### Почему это происходит

**React + useReducer** работают по принципу **иммутабельности**:
- Если объект мутирует (изменяется внутри), React **не видит** изменений
- Нужно создавать **новый объект** (копию), чтобы React обнаружил изменения
