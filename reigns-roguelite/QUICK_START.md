## 5.2. ЧТО ИСПРАВЛЕНО В v0.7.2 (24 марта 2026, вечер)

### ✅ Критические баги геймплея

**1. Побег без штрафов:**
- **Проблема:** Игрок сбежал до первого клиента — 0 потерь
- **Причина:** Формула рассчитывала штраф от 0 (не было ходов)
- **Решение:** 15% штраф от денег хаба если 0 обслуженных клиентов

**2. Special/intense карты на 1 уровне:**
- **Проблема:** Игрок 1 уровня получает "Легендарное действие"
- **Причина:** Нет ограничений по уровню на intensity карт
- **Решение:** Special < 5 уровня, Intense < 3 уровня заблокированы

**3. Карты с предметами без предметов:**
- **Проблема:** Карта "С предметом" генерируется без предметов
- **Причина:** `player.inventory` не существует (undefined)
- **Решение:** Проверка `player.items || player.inventory || []`

**4. Завершение клиента завершает ночь:**
- **Проблема:** Выбор исхода кульминации → конец ночи
- **Причина:** Это фича! После climax → следующий клиент
- **Решение:** Проверено: работает корректно (climax → следующий клиент → ночь)

**Исправленные файлы:**
- `src/engine/GameEngine.js` — `endNightEarly()`: штраф от хаба если 0 клиентов
- `src/system/ActionDeckSystem.js` — `_getAvailableActions()`: level restrictions, item check
- `scripts/test.js` — адаптирован под новую логику штрафов

## 5.4. ЧТО ИСПРАВЛЕНО В v0.8.1 (24 марта 2026, ночь)

### ✅ Критические баги (тестирование)

**1. totalReward ошибка:**
- **Проблема:** `Cannot access 'totalReward' before initialization`
- **Причина:** edgeStackBonus использовал totalReward до объявления
- **Решение:** Сначала расчёт totalReward, потом edgeStackBonus

**2. Arousal застревает на 92-94%:**
- **Проблема:** t_ease карты сбрасывали arousal дважды за ход
- **Причина:** Нет флага что edge stacking уже применён
- **Решение:** Флаг `_edgeStackAppliedThisTurn`

**3. attemptEarlyFinish не работает:**
- **Проблема:** `success: true` даже при неудаче
- **Причина:** Неправильный return при неудаче
- **Решение:** `success: false` при неудаче

**4. Карты не зависят от фазы:**
- **Проблема:** Генерация карт использовала старый arousal
- **Причина:** `client.arousal` из `clientData` не обновлялся
- **Решение:** `sessionClient` с актуальным arousal из сессии

**Исправленные файлы:**
- `src/core/InteractionSession.js` — totalReward, edge stacking флаг, attemptEarlyFinish
- `src/engine/GameEngine.js` — sessionClient для генерации карт

## 8. КОНТЕКСТ

### Игровой цикл v0.7
```
Хаб → Кнопка "Gloryhole Quest" →
Карта клиента + 2 карты действий →
Использовать действия (arousal растёт) →
Смена фазы (цвет) → Завершить ночь →
Возврат в хаб → Отдохнуть → Новый день
```

### actionPoints (очки действий)
- **3 очка** базово в день
- **+0-2 очка** от навыка "Выносливость" (5 и 10 уровень)
- **Тратятся** на действия в хабе (🧘, 🔞, 🐱, 📱)

### Прокачка навыков
```
1 уровень репутации = 1 очко навыка
1 очко → +1 уровень любого доступного навыка
```

### Требования для навыков
- **Глубокий**: Оральный 5+
- **Дразнение**: Наблюдательность 3+, Выносливость 2+
- **Доминантность**: Репутация 5+, Наблюд. 5+, Внешность 5+

## 📝 ЗАМЕТКИ НА БУДУЩЕЕ

**Важно помнить:**
- React требует **иммутабельности** — всегда создавай копии объектов
- EventBus + useReducer = нужна подписка на все события
- `{ ...object }` создаёт поверхностную копию (для вложенных нужен `structuredClone`)

**Правила для GameEngine:**
1. `getActionState()` возвращает **копии** объектов
2. `getGameState()` возвращает **копии** объектов
3. `getHubState()` возвращает **копии** объектов

**Причина:** React не видит мутаций, только новые объекты!
